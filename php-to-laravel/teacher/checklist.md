# レビュー観点・つまずき所・採点／PHP→Laravel 移行

## 完成の確認（機能が素PHP版と同等か）
- [ ] ログイン/ログアウト（bcrypt。SQLiでログインできない）
- [ ] 一覧・検索・並び替え・ページング（検索条件がページ送りで維持される）
- [ ] 登録・編集・削除・画像アップロード
- [ ] バリデーション（登録・編集の両方、メール重複）
- [ ] CSV（文字化けしない）・登録メール（届く）

## 移行の“質”の確認
- [ ] SQLが Eloquent/クエリビルダで、生SQL連結が残っていない
- [ ] Blade の `{{ }}` でエスケープ（`{!! !!}` を安易に使っていない）
- [ ] `@csrf` が全フォームにある
- [ ] マジックナンバーが定数化されている
- [ ] N+1 が解消されている

## よくあるつまずき
- **`laravel-legacy` をそのままコピー**してバグごと持ち込む → 構造参考に留め、正しく書かせる。
- 認証で `users` の md5 をそのまま照合しようとする → シーダーごと bcrypt に作り直すのが正解。
- 一覧の検索 OR をグルーピングし忘れる（素PHPと同じ罠）。
- 並び替えでユーザー入力をそのまま `orderBy` に渡す → ホワイトリスト必須。
- 画像で `enctype` / `storage:link` を忘れる。
- ページャで `appends()` を忘れて条件が消える。

| 観点 | 配点 |
